# 1. Product Overview

## 1.1 What Is Estrogen Pharmacy

Estrogen Pharmacy is a digital-first, women-exclusive pharmacy platform operating in Saudi Arabia. It provides a mobile application (iOS and Android) and a web-based administration portal that enables women to browse, order, and receive pharmacy products — including over-the-counter (OTC) medications, prescription medications, personal care, maternal health, reproductive health, and chronic condition management products — in a private, trusted, and Arabic-first environment.

The platform connects three operational layers:
1. **Customer Layer** — women browsing and purchasing via mobile app
2. **Pharmacist Layer** — licensed pharmacists reviewing prescriptions, approving orders, and consulting patients
3. **Admin Layer** — operations team managing inventory, catalog, users, promotions, and system configuration

The backend integrates with the pharmacy's Point of Sale (POS) and inventory management system via API to maintain real-time or near-real-time stock accuracy.

---

## 1.2 Who It Serves

Estrogen Pharmacy serves:
- Women of all ages in Saudi Arabia — from teenage girls to elderly women
- Women managing chronic conditions requiring ongoing pharmacy access
- Pregnant and postpartum women with specialized product needs
- Caregivers (mothers, sisters, daughters) ordering on behalf of female family members
- Women who require privacy when accessing sensitive health products

The platform is Arabic-first, with English as a fully supported secondary language.

---

## 1.3 Why It Exists

Saudi Arabia's pharmacy sector is largely gender-neutral, operated in public-facing physical stores where women — particularly those seeking sensitive health products (menstrual care, hormonal medications, fertility products, postpartum supplies, intimate health products) — often experience:
- Discomfort purchasing sensitive items from male pharmacists in public
- Limited access to women-specific health education in-store
- No digital alternative purpose-built for their needs
- Generic pharmacy apps that treat women's health as a subcategory, not a core offering

Estrogen Pharmacy fills this gap with a platform designed entirely around women's dignity, privacy, and health continuity.

# 4. User Roles and Permissions

## 4.1 User Role

A User is any woman who registers an account on the Estrogen Pharmacy mobile app.

**Authentication:**
- Register with phone number (OTP) or email + password
- Biometric login (Face ID / fingerprint) supported after first login
- Guest browsing allowed; prescription checkout and account features require registration

**Account Management:**
- Edit personal details (name, date of birth, phone, email)
- Manage delivery addresses (multiple addresses, labeled: Home, Work, Other)
- Set language preference (Arabic / English)
- Set notification preferences (push, SMS, email — per type)
- View and manage saved payment methods
- Set account PIN for additional security

**Health Profile:**
- Add known allergies
- Add medical notes (optional, private field)
- Upload and store prescription documents
- Set pregnancy or breastfeeding status (affects displayed medication warnings)
- Chronic condition tags (optional, self-declared, used for product filtering)

**Shopping:**
- Browse product catalog (categories, search, filters)
- View product detail pages including safety warnings, ingredients, storage
- Add items to cart
- Apply promo/coupon codes
- View delivery fee calculation before checkout
- Place OTC orders directly
- Place prescription orders (upload required → pharmacist review → fulfillment)
- View personalized recommendations (Phase 2+)

**Orders:**
- View full order history
- View real-time order status
- Request cancellation (before pharmacist approval or packing)
- Request refund or return (per policy)
- Reorder from previous orders
- Rate and review an order (Phase 2)

**Communication:**
- Contact support via in-app help center
- Submit support ticket
- Chat with pharmacist (Phase 2)
- View in-app notifications
- Receive SMS/push order and health alerts

**Loyalty (Phase 2+):**
- View loyalty points balance
- Redeem points at checkout
- View referral code and referral history

---

## 4.2 Pharmacist Role

A Pharmacist is a licensed pharmacy professional onboarded by admin. They access a dedicated dashboard (browser-based, tablet-optimized).

**Authentication:**
- Email + password login
- 2FA enforced (TOTP or SMS OTP)
- Session timeout after inactivity

**Prescription Queue:**
- View all pending prescriptions assigned to their queue
- Open and review uploaded prescription image/PDF
- View patient profile information relevant to the order (allergies, medical notes, declared health conditions)
- View full product list in the order
- Cross-check prescription against ordered items

**Order Processing:**
- **Approve** prescription: marks order as verified, triggers packing workflow
- **Reject** prescription: must provide reason; patient notified
- **Hold** prescription: request additional information from patient
- **Substitute** medication: suggest alternative SKU if original is out of stock (must be within allowed substitution rules; must be validated with compliance advisor)
- Add pharmacist notes to an order (internal and patient-visible notes available)
- Escalate order to admin if special handling required

**Patient Communication:**
- Send in-app message to patient requesting prescription clarification (Phase 2)
- View past communication for a given order
- Cannot initiate outbound contact to patients outside of order context

**Inventory Visibility:**
- View current stock levels for products they are processing orders for
- Cannot edit inventory, pricing, or product catalog
- Can flag a product as "stock concern" for admin to review

**Analytics (Limited):**
- View their own queue metrics: prescriptions reviewed per day/week, average review time
- Cannot access revenue data, marketing analytics, or full user data

**Restrictions — Cannot:**
- Access user payment information
- Access admin settings or system configuration
- Modify product catalog, pricing, or promotions
- Access other pharmacists' queues (unless authorized by admin)
- Delete orders, users, or any permanent records
- Access audit logs of other roles

---

## 4.3 Admin Role

Admin has full access to all system functions. There may be sub-admin roles with scoped permissions (e.g., Operations Admin, Catalog Admin, Finance Admin) configurable from the role management panel.

**User Management:** Create, view, edit, disable, or delete user accounts. Add support notes. Review flagged accounts.

**Pharmacist Management:** Add, edit, deactivate pharmacist accounts. Track license expiry. Assign pharmacists to branches (multi-branch, Phase 4). Monitor workload and performance.

**Catalog Management:** Full CRUD on products, categories, brands, SKUs, pricing, images, and content. Toggle prescription-required flag. Manage product bundles and kits.

**Order Management:** View all orders across all states. Override order status. Process manual cancellations and refunds. Handle escalated orders from pharmacists.

**Inventory:** View synced stock from POS. Set manual overrides. Configure low-stock alert thresholds. Review sync logs. Trigger manual resync.

**Content:** Manage banners, featured collections, health articles, FAQs, policy pages.

**Promotions:** Create and manage coupons, campaigns, loyalty rules, referral programs.

**Analytics:** Full access to all dashboards — sales, retention, product performance, pharmacist metrics, sync health, conversion.

**Settings:** Configure delivery zones, payment gateways, notification templates, tax rules, API credentials, language configuration, privacy settings.

**Audit and Security:** View full audit log of all system actions. View role change history. Configure access control rules.

# 5. Core Features for the User App

## 5.1 Onboarding and Authentication

### Signup Flow
1. User opens app → sees Arabic-first welcome screen with brand imagery
2. Tap "إنشاء حساب" (Create Account)
3. Enter mobile number → OTP sent via SMS (6-digit, 90-second expiry)
4. OTP verified → set password (minimum 8 characters, 1 uppercase, 1 number)
5. Enter name (first and last)
6. Confirm date of birth (used for age-appropriate content)
7. Accept terms of service and privacy policy (explicit tap-to-accept, no pre-checked boxes)
8. Optional: declare pregnancy or chronic condition status (can be skipped and set later)
9. Account created → redirected to home

### Login Flow
- Phone number + password
- Alternatively: email + password (after email is added to profile)
- OTP login option (for users who forget password or prefer OTP)
- Biometric login (Face ID / fingerprint) enabled on first successful login if device supports it

### Guest Browsing
- Users can browse catalog without registration
- Adding to cart, viewing pricing, and reading product details allowed
- Checkout requires registration (soft gate: prompt to register at checkout)
- Prescription products cannot be viewed in detail without registration

### Password Reset
- Enter phone number → OTP → set new password
- Alternative: email-based reset link (if email on file)

### OTP Rules
- OTP is 6 digits, expires in 90 seconds
- Max 3 OTP attempts per session before 15-minute lockout
- Rate limiting: max 5 OTP requests per phone number per hour

### Session Management
- JWT-based sessions
- Access token: 15-minute expiry
- Refresh token: 30-day expiry, rotated on use
- Force logout all sessions option available in account settings
- Suspicious login detection triggers re-authentication (new device, new location)

## 5.4 Medication Ordering

### OTC Products
- Add to cart directly from product page or listing
- No prescription required
- Standard checkout flow applies

### Prescription Products
- Marked with "يتطلب وصفة طبية" (Requires Prescription) badge
- User can add to cart but cannot checkout without uploading prescription
- Prescription upload prompt triggered at checkout if not uploaded
- System accepts: JPG, PNG, PDF; max 10MB per file
- Multiple prescription images can be merged per order

### Prescription Upload Workflow
1. User selects prescription product → adds to cart
2. At checkout: system detects prescription-required items → prompts upload
3. User uploads prescription image/PDF
4. Order placed in "Pending Pharmacist Review" state
5. Pharmacist reviews → approves, rejects, or holds
6. User notified of decision via push notification + SMS
7. If approved: order moves to packing/fulfillment
8. If rejected: user notified with reason; can resubmit
9. If held: user receives message requesting clarification

### Prescription Validation Rules
- Prescription must be legible (pharmacist manually reviews)
- Prescription must match ordered product (pharmacist verifies)
- Prescription must not be expired (pharmacist checks date)
- Prescription must include: physician name, patient name, date, clinic/hospital stamp
- Maximum quantity per prescription order: must be validated with compliance advisor
- Controlled substances: must be validated with SFDA and compliance advisor

### Reorder Logic
- "Reorder" button on any past order or past prescription order
- System checks current stock availability before adding to cart
- If same product unavailable → "Product Unavailable" notice + suggested alternative
- If prescription was previously verified: reorder still requires pharmacist review for prescription products (regulatory requirement — must be validated with compliance advisor)

### Substitution Rules
- Pharmacist can suggest a substitution if original product is out of stock
- Patient receives notification: "Pharmacist is suggesting an alternative: [Product Name]. Do you accept?"
- Patient must explicitly accept or decline the substitution
- Declining returns the order to held state for pharmacist review
- Substitution must be medically equivalent — pharmacist responsibility
- Admin can define approved substitution pairs in the catalog

## 5.5 Cart and Checkout

### Cart
- Persistent cart (saved across sessions)
- Maximum 50 distinct line items per cart
- Real-time stock validation on cart open (items flagged if gone out of stock)
- Prescription items visually distinguished from OTC items in cart
- Bundle discount applied automatically if applicable
- Remove item, update quantity inline

### Checkout Steps
1. Review cart
2. Select or add delivery address
3. Select delivery option (standard / express / scheduled)
4. Apply promo code (single code per order)
5. Review order summary (subtotal, delivery fee, discount, VAT, total)
6. Select payment method
7. Confirm order (explicit "Place Order" tap)

### Delivery Fee Logic
- Fee based on delivery zone (admin-configured by zone)

## 5.6 Delivery and Tracking

### Delivery Types
- **Standard Delivery:** 2–4 hours or same-day/next-day depending on zone (admin-configured)
- **Express Delivery (Phase 2):** 60-minute delivery in eligible zones
- **Scheduled Delivery (Phase 2):** User selects preferred delivery date and time window

### Discreet Packaging
- Default: unbranded outer packaging for all orders
- Option to request "discreet packaging" label on account or per-order basis
- No product names or pharmacy branding visible on outer packaging
- Inner packaging follows standard pharmacy labeling requirements

### Order Status States
1. **Order Placed** — payment confirmed, awaiting pharmacist review (if prescription) or fulfillment
2. **Pharmacist Review** — prescription under review (prescription orders only)
3. **Approved** — prescription verified; packing in progress
4. **Ready for Pickup by Courier** — packed and awaiting courier
5. **Out for Delivery** — courier has picked up
6. **Delivered** — confirmed delivered
7. **Cancelled** — order cancelled before packing
8. **Refund Requested** — refund/return initiated
9. **Refunded** — refund processed

### Live Tracking
- Status bar in Orders section
- Push notification at each status change
- Estimated delivery time shown
- Courier contact option (Phase 2)

### Failed Delivery Handling
- If delivery attempt fails: courier leaves attempt notification
- User is notified → can reschedule or select pickup
- 2 failed attempts: order held for 24 hours; if unclaimed, return to pharmacy and refund processed
- Admin and ops team notified of failed delivery events
